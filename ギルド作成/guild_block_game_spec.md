# 1. ゲームタイトル

**「ギルド抗争：覚醒ブロック戦線」**

---

# 2. ストーリー

ガイザー災害後――  
異世界エネルギーにより人類は「覚醒」という新たな力を手に入れた。

しかしその力は希望だけでなく、狂気も生み出した。  
覚醒に溺れた者たちは犯罪者となり、世界は無秩序へと崩壊する。

警察も政府も機能しない中、  
人々は自らを守るため「ギルド」を結成する。

プレイヤーはギルドのリーダーとなり、  
覚醒犯罪者たちとの知略戦――  
**ギルド戦闘**に挑む。

### テーマと遊びの対応（フレーバー）

| ゲーム上 | 世界観での意味（演出・文言用） |
|----------|-------------------------------|
| 青ブロック | ギルドが確保する「安定領域・結界の石」 |
| 赤ブロック | 犯罪者側の「暴走した覚醒の塊」 |
| ライン5 | 陣地の一本化／敵勢力の連携ライン |
| 高さ10赤3個 | 覚醒エネルギーを天井へ3点突破し制圧 |

リザルトやチュートリアルで上記を1文ずつ入れると、抽象ルールと世界観がつながる。

# 3. ゲーム概要

- ジャンル：**戦略パズル（拡張〇×ゲーム / 五目並べ + 積み上げ）**
- プレイ時間：3〜5分
- プラットフォーム：Webブラウザ（HTML/CSS/JavaScript）
- 対戦形式：**CPU戦 / 対人戦**（ターン制）
- **先手**：青（ギルド）固定（CPU/対人で共通）
- CPU戦：赤（犯罪者）は簡易AI
- 対人戦：青・赤とも人間（交互に配置）

# 4. ゲームの流れ

## ① タイトル画面

- ゲームタイトル表示
- 「CPU戦」ボタン
- 「対人戦」ボタン
- 「ルール説明」ボタン（勝利条件・配置ルール・斜めの扱いを要約表示）

## ② プレイ画面

- 共通：ターン制で交互にブロック設置（§7の合法手に従う）
- ブロックは**下から積み上げ**、または**同一段上で横に隣接するマス**へ配置可能（詳細は §7）
- CPU戦：青（ギルド）が人間、赤（犯罪者）がAI
- 対人戦：青・赤とも人間（2人で交互に配置）

### ターン処理

1. 手番プレイヤーが配置
2. 勝利・膠着判定
3. 次の手番へ移行（CPU戦：赤はAI／対人戦：相手プレイヤー）

## ③ 決着

- 勝利：ギルド勝利（犯罪者討伐）
- 敗北：犯罪者支配
- 膠着：§5 参照（この場合はギルド勝利として扱う）
- リザルト画面へ遷移

## ④ リザルト画面

- 勝敗（または膠着）表示
- ターン数表示
- 「RETRY」ボタン

# 5. 勝敗条件

## 勝利条件

この対戦は「防衛（ギルド）vs 攻撃（犯罪者）」として目標が非対称。  
勝利条件を先に満たした陣営が勝ち。

1. **防衛成功：ギルド（青）のライン勝ち**  
   青ブロックが、**直線上に連続して5個**並ぶ。  
   - **縦**：同一列で段が連続  
   - **横**：**同一段**で列が連続  
   - **斜め**：盤面の斜め2方向（／ と ＼）  
   - **立体斜め**は**ライン勝ちに含めない**

2. **陥落達成：犯罪者（赤）の高さ占拠勝ち**  
   赤ブロックが、**10段目（最上段）**にある状態が**3個以上**そろう。

## 膠着（防衛勝利）

- 合法手がなく、勝利条件が未達の場合は、**ギルド（青）の防衛成功**として扱う。

# 7. 主要要素

## フィールド

- グリッド：**10列 × 10段**（列＝横位置、段＝底から数えた高さ）
- 各マスは「その列における積み上げの一段」を表す。下から順に埋まる。

## キャラクター

### プレイヤー（ギルド）

- 青ブロック
- 秩序を守る存在

### 敵（犯罪者）

- 赤ブロック
- ルールベースの簡易戦略AI（§11）

---

## 配置ルール（合法手）

- **初手**：**最下段（1段目）**のマスにのみ配置可能。空いている列の底に1個置くイメージ。
- **2手目以降**：次の**いずれか**を満たす空マスにのみ配置可能。  
  1. **下支え**：そのマスの**真下**（一段下、同一列）にブロックがある。  
  2. **横隣接**：**同一段**にあり、左右（東西）のいずれかに**いずれかの色のブロック**が隣接している空マス。

### 配置ルールの例（イメージ）

- 列Aの3段目にブロックがある → 同じ列Aの4段目は「下にブロックがある」ので空いていれば合法。  
- 列Bの2段目にブロックがあり、列Cの2段目もブロックがある → **同じ2段目**でBとCが左右隣なら、お互いの横方向の空きは「横隣接」で繋がりやすい。  
- **一段上の斜め**だけで繋がっているマスは、上記2条件を満たさない限り**不可**。



# 8. 成長・強化要素（簡易）

※短時間ゲームのため軽量設計

- 連勝数に応じた「スキン変化」
- 特殊演出解放（エフェクト）

**データ保存**：ゲーム本編のセーブは行わない。**連勝数・解放フラグのみ** `localStorage`（または `sessionStorage` に限定する運用も可）で保持する。未保存の場合は連勝0扱いでよい。

# 11. 技術仕様

## 描画方式

- HTMLの `div` グリッドで構築
- CSS Grid 使用

## ゲームループ

- ターン制（常時ループは不要）
- 入力 → 合法手チェック → 状態更新 → 描画 → 勝敗判定

## 勝利判定ロジック（配置直後）

- **犯罪者勝利（赤）**：配置後、10段目（最上段）に赤ブロックが3個以上そろっているか。
- **ギルド勝利（青）**：配置後、青が直線上に連続5個を作っているか（縦・横・斜め2方向）。

## AI仕様（簡易・優先順位）

1. **即勝ち**：自分が（ギルドなら）ライン勝ち、（犯罪者なら）「10段目赤3個」を達成できる手があればそれを打つ。  
2. **即負け阻止**：相手が次の一手で勝てるマスがあれば、可能ならそこを塞ぐ／奪う（複数ある場合はランダムでよい）。  
3. **脅威拡大**：自分の「連続4」「両端空き3」など、次々手で勝ちやすい手を簡易スコアで選ぶ（実装が重ければ「3連を延ばす手」をランダムに選ぶ程度でも可）。  
4. **ランダム**：上記以外は合法手からランダム。

# 12. 実装ポイント

- 配置可能マスをハイライト表示
- クリック／タップ時に即反映（非法手は無視 or トースト）
- 勝利時にライン演出（斜め・横は同一段であることを前提にしたハイライトでよい）
- スマホ対応（タップ領域の十分な余白、ズーム抑止など）
- ルール説明画面に「斜めは同一段のみ」「立体斜めは不可」を明記

---

# 13. 面白さのコア

- **シンプル × 戦略性**：ルールは短く、読みは深くなる余地を残す。
- 五目並べ＋積み上げによる立体感（防衛のライン5と、攻撃の「高さ10に赤3個」）。
- 攻撃側は「高さ10へ3点突破」を狙うため、単なる横展開だけでは止められない。
- **短時間でも駆け引きが熱い**：AIに阻止・延長線の読みを持たせる（§11）。
